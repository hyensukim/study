0. 인터넷(Internet)이란?
- Inter-network의 준말이며, "통신망을 하나로 연결한다"는 의미이다.
- 현대에는 "전 세계 컴퓨터들을 하나로 연결하는 거대한 컴퓨터 통신망"을 의미한다.
- 즉, 인터넷이란 수많은 클라이언트 컴퓨터와 서버 컴퓨터, 그리고 이들로 구성된 네트워크의 집합체이다.
* 네트워크 : 컴퓨터 간의 연결

1. 웹(WEB)이란?
- WWW 혹은 W3라고도 부르며, World Wide Web을 줄여서 '웹'이라고 한다.
- 인터넷에 연결된 사용자들이 서로의 정보를 공유할 수 있는 공간.
- 인터넷과 같은 의미 같지만, 정확히 말하면 인터넷 상의 인기 있는 서비스 중 하나이다.

2. 웹의 특징
- 웹은 인터넷 상에서 텍스트, 그림, 소리, 영상과 같은 멀티미디어 정보를 문서에 담아 하이퍼텍스트 방식으로 연결하여 제공함.
* 하이퍼 텍스트란?
    - 문서 내부에 다른 문서로 연결되는 참조를 넣음으로서 웹 상의 여러 문서들이 서로 참조할 수 있도록 하는 기술.
    - 문서 내부에 다른 문서로 연결되는 참조를 '하이퍼 링크'라고 한다.
- 여기서 문서는 HTML, CSS, JS 등을 이용해 만든 웹페이지를 의미한다.
- 이러한 웹페이지에는 HTTP라는 프로토콜을 사용하면 누구나 접근하고 사용할 수 있게 된다.

3. 웹의 구성
- 웹에서 HTML 언어를 사용하여 작성된 하이퍼텍스트 문서를 웹페이지라고 한다.
- 이러한 웹페이지들이 서로 관련된 내용으로 작성된 집합을 웹사이트라고 한다.
- 웹은 이렇게 작성된 수많은 웹 페이지들이 하이퍼 링크를 통해 연결되어 구성한다.

4. 웹 서핑
- 사용자가 웹 페이지에 포함된 하이퍼링크를 따라 다른 웹 페이지들로 계속하여 이동하는 것을 웹 서핑(web surfing)이라고 부르며,
  이때 사용자가 웹 페이지를  검색하기 위해 사용하는 프로그램을 웹 브라우저(web browser)라고 한다.

* 명령어 집합 -> 소프트웨어 = 프로그램 = 어플리케이션 -> 실행 -> 프로세스

5. 인터넷 시스템
- 인터넷을 구성하고 있는 시스템.
- 이러한 시스템은 여러 서버와 클라이언트로 구성되어 있다.
- 서버 ? 사용자에게 인터넷 서비스를 제공하는 컴퓨터.
- 클라이언트 ? 사용자 혹은 사용자가 사용하는 기기 ex) 사용자 컴퓨터에 웹브라우저, 사용자의 핸드폰 기기 등...

6. 웹 서버
- 사용자가 요청하는 웹페이지 또는 프로그램을 실행하여 해당 파일 혹은 그 결과를 사용자에게 제공함.
- 사용자 요청에 따라 항시 웹서비스를 제공하기 위해 웹서버(컴퓨터)는 항상 돌아간다.

7. 웹 브라우저
- 웹브라우저는 웹 서버에서 웹 페이지를 가져오거나 웹 서버로 정보를 보낼 때 사용하는 프로그램.
- 웹브라우저는 사용자가 웹페이지를 요청하면 웹서버로 부터 데이터를 불러와서 사용자 화면에 출력해준다.

8. 프로토콜(protocole)
- 컴퓨터끼리 정보를 주고받기 위한 일종의 규칙이나 약속을 의미한다.
- 컴퓨터끼리 서로 원활한 통신(데이터 교환)을 위해 데이터 형식이나 통신 순서를 미리 일치시켜야 한다.
- 이렇게 컴퓨터 사이의 통신에서 정보를 교환하기 위해 미리 만들어 둔 네트워크 통신 규약을 프로토콜이라고 부른다.

9. 인터넷 주소
- 인터넷 상에서 통신하고 싶은 컴퓨터 혹은 서버 컴퓨터를 찾기 위해서는 IP 주소 또는 도메인 네임 중 하나를 알고 있어야 한다.
- 정확히 말하면 컴퓨터의 주소는 IP 주소를 의미하며, 도메인 네임은 결국 IP주소로 변환된다.

10. IP 주소
- 인터넷 상에는 수많은 컴퓨터와 서버간의 연결이 이루어져 있다.
- 이러한 인터넷 속에서 컴퓨터를 식별하기 위해 지정된 번호가 IP 주소이다.
- 우편 배달부가 우편을 전달하기 위해 집 주소로 찾는 것과 같다.
- 현재 사용되고 있는 IP 주소 체계는 IPv4(IP 버전4)로 0부터 255까지의 십진수 네 개를 점(.)으로 구분하여 사용한다.
- 32비트의 주소 체계인 IPv4는 이론상 약 43억 개의 IP 주소를 나타낼 수 있지만, 인터넷의 급격한 발달에 따라 IP 주소가
  부족해지는 현상이 발생한다.
  ex) 202.179.177.22 ... 
- 따라서 이에 대한 해결책으로 고안된 128비트의 주소 체계인 IPv6(IP 버전6)의 사용이 점점 증가되고 있는 추세이다.

11. 도메인 네임
- IP 주소는 사람이 기억하기 어려우므로, 이를 쉬운 문자형태로 표현한 것을 도메인 네임이라고 한다.
- 도메인 네임은 naver.com과 같이 의미있는 문자들과 점으로 구성된다.

12. 웹의 작동 원리
    1. 사용자가 웹 브라우저를 통해 찾고 싶은 웹 페이지의 URL 주소를 주소창에 입력함.

    2. 사용자가 입력한 URL 주소 중에서 도메인 네임(domain name) 부분을 DNS(도메인네임시스템) 서버에서 검색함.

    3. DNS 서버에서 해당 도메인 네임에 해당하는 IP 주소를 찾아 사용자가 입력한 URL 정보와 함께 전달함.

    4. 웹 페이지 URL 정보와 전달받은 IP 주소는 HTTP 프로토콜을 사용하여 HTTP 요청 메시지를 생성함.

    5. 이렇게 생성된 HTTP 요청 메시지는 TCP 프로토콜을 사용하여 인터넷을 거쳐 해당 IP 주소의 컴퓨터로 전송됨.

    6. 이렇게 도착한 HTTP 요청 메시지는 HTTP 프로토콜을 사용하여 웹 페이지 URL 정보로 변환됨.

    7. 웹 서버는 도착한 웹 페이지 URL 정보에 해당하는 데이터를 검색함.

    8. 검색된 웹 페이지 데이터는 또 다시 HTTP 프로토콜을 사용하여 HTTP 응답 메시지를 생성함.

    9. 이렇게 생성된 HTTP 응답 메시지는 TCP 프로토콜을 사용하여 인터넷을 거쳐 원래 컴퓨터로 전송됨.

    10. 도착한 HTTP 응답 메시지는 HTTP 프로토콜을 사용하여 웹 페이지 데이터로 변환됨.

    11. 변환된 웹 페이지 데이터는 웹 브라우저에 의해 출력되어 사용자가 볼 수 있게 됨.
